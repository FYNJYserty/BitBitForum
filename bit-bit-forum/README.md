### Общая информация
Этот проект разрабатывался в качестве дипломной работы и как пет-проект. 
Основной целью было изучить и освоить на практике современный стек технологий,
применить на практике теоретические знания в области клиент-серверных приложений,
микросервисной архитектуре. Также возникла потребность изучить Spring Framework
и сопутствующие ему модули.
Проект разрабатывался полность одним разработчиком, поэтому могут быть недочёты.
### О проекте
BitBitForum - форум для общения, в который были включены дискуссии, личные чаты пользователей и 
каналы по аналогии с именитыми мессенджерами. Есть несколько встроенных мини игр,
в качестве микросервиса был придуман модуль отправки уведомлений через электронную почту.

### Как запустить
Локально проект можно запустить через заранее установленный Docker на своём 
компьютере, но для этого следует извлечь проект из GitHub. Последовательность команд запуска 
приведена ниже.

1. Скопировать Git репозиторий на свой компьютер
```bash
git clone https://github.com/FYNJYserty/BitBitForum
```
2. Перейти в каталог проекта
```bash
cd BitBitForum
```
3. Запустить контейнеры при помощи Docker Compose
```bash
docker-compose up -d
```
4. Дождитесь окончания сборки и запуска всех сервисов
5. Проверьте статус контейнеров
```bash
docker-compose ps
```
6. После успешного запуска приложение будет доступно в браузере по адресу
```bash
http://localhost:8080/
```
7. Просмотреть логи внутри контейнера выполните команду
```bash
docker-compose logs -f 
```
8. Остановить работу контейнеров по команде
```bash
docker-compose down 
```
Данного набора команд достаточно для запуска проекта.

Примечание: Убедитесь, что Docker и Docker Compose установлены на вашем компьютере.
Если проект использует специфические порты, убедитесь, что они свободны.
